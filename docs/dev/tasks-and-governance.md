# Tasks and Governance

## Task Tracking

- Active and pending work is tracked in `TASKS.md`.
- Completed context may be moved to `archival_tasklist.md`.

## Governance Files

- `AGENTS.md`: project rules and execution constraints
- `TASKS.md`: ordered implementation queue and current priorities
- `archival_tasklist.md`: historical completed tasks retained with enough structured detail for reconstruction

## Workflow Expectations

- Start from the first pending task unless explicitly redirected.
- Keep report exclusions and review-driven lists synchronized with current decisions.
- Prefer cache-backed rebuild behavior for external-source pipelines.
- Avoid destructive or unrelated repo changes during focused tasks.
